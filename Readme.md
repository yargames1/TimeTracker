# Time Tracker Application

## Описание проекта

Time Tracker — это backend-приложение для отслеживания задач и учета рабочего времени сотрудников.

Приложение позволяет:

* создавать и управлять задачами
* отслеживать время, затраченное на выполнение задач
* получать записи рабочего времени за выбранный период

Проект реализован как REST API с использованием Spring Boot и MyBatis.

---

## Технологии

* Java 17
* Spring Boot
* MyBatis
* H2 Database (in-memory)
* JUnit 5, Mockito (для тестирования)
* Maven

---

## Функционал

### Работа с задачами

* Создание задачи
* Получение задачи по ID
* Обновление статуса задачи

Статусы:

* NEW
* IN_PROGRESS
* DONE

---

### Учет рабочего времени

* Создание записи времени (TimeRecord)
* Привязка записи к задаче и сотруднику
* Получение записей за период

---

## API эндпоинты

### Tasks

#### Создать задачу

POST /tasks

```json
{
  "title": "Test task",
  "description": "Description task"
}
```

---

#### Получить задачу

GET /tasks/{id}

---

#### Обновить статус

PATCH /tasks/{id}/status

```json
{
  "status": "DONE"
}
```

---

### Time Records

#### Создать запись времени

POST /time-records

```json
{
  "employeeId": 1,
  "taskId": 1,
  "startTime": "2026-04-30T10:00:00",
  "endTime": "2026-04-30T12:00:00",
  "description": "Work session"
}
```

---

#### Получить записи за период

GET /time-records?employeeId={id}&start={start_time}&end={end_time)

## Тестирование

Для "ручного" тестирования в корне представлен каталог postman коллекцией тестов
```
postman/collections/Tests
```

В проекте реализованы unit-тесты для сервисного слоя:

* проверка бизнес-логики
* обработка ошибок
* взаимодействие с DAO (через Mockito)

Для запуска тестов:

```bash
mvn test
```

# Планируемые изменения

Ввиду ограниченного времени работы не весь функционал реализовать удалось, но данные изменения планируются для того, чтобы API можно было использовать в работе

* Таблица с сотрудниками и их регистрация
* Вывод всех активных задач / завешенных задач / новых задач
* Изменение формата отчетности о работе (взял задачу, отчитался о начале работы -> закончил задачу, отчитался о выполнении)
* Более точная обработка ошибок
